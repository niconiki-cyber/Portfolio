My Process

How I Work

A structured approach that ensures every project is delivered with quality and clarity.

01.

Discovery & Research

I begin by understanding the product requirements, backend APIs, and user flows. This includes reviewing specifications and mapping out all integration points before writing a single test case.

02.

Planning & Documentation

I design flowcharts, create structured test case documents, and plan the scope of testing. Clear documentation ensures the whole team stays aligned throughout development.

03.

Execution & Testing

Running API tests in Postman, validating response codes and data, testing edge cases, and verifying mobile UI behavior across different states and scenarios.

04.

Review & Delivery

Bug reporting, retesting fixes, and final sign-off. Ensuring every module — from login and OTP to orders and wallet — functions exactly as specified.
